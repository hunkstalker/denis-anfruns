Si cargas múltiples colecciones en el evento `App.OnStart` una detrás de otra con `;`, estás perdiendo tiempo. La función `Concurrent` evalúa todas las fórmulas al mismo tiempo.

<CodeTabs 
  items={[
    { 
      label: 'Power Apps',
      code: `// ❌ Lento (secuencial)
ClearCollect(Usuarios, Office365Users.SearchUser());
ClearCollect(Departamentos, ["IT", "HR", "Ventas"]);

// ✅ Rápido (paralelo)
Concurrent(
    ClearCollect(Usuarios, Office365Users.SearchUser()),
    ClearCollect(Departamentos, ["IT", "HR", "Ventas"])
);`, 
      lang: 'power fx' 
    }
  ]}
/>

Esto reduce el tiempo de inicio de la aplicación drásticamente, especialmente cuando conectas con múltiples fuentes de datos.

Es ideal para cargas iniciales (`OnStart`) o botones que lanzan múltiples procesos independientes.

<Callout type="warning" title="Ojo con el separador" filled>
  Recuerda que si tienes Power Apps configurado para la **región europea** tendrás que cambiar la coma `,` por punto y coma `;` en las fórmulas. Y Doble punto y coma `;;` para saltos de línea donde normalmente se pondría únicamente un punto y coma `;`.
</Callout>
